# 0G Tokenomics Agent

> AI-powered inference dashboard for the [0G Compute Network](https://0g.ai) — submit on-chain AI jobs, explore decentralized providers, and get expert Web3 tokenomics analysis.

🌐 **Live demo:** [zero-cup.vercel.app](https://zero-cup.vercel.app)

---

## What It Does

**0G Tokenomics Agent** bridges users with the 0G Compute Network — a decentralized AI inference marketplace. Submit prompts through on-chain 0G providers or external LLMs (Groq, OpenAI, Anthropic, DeepSeek, and more), track provider availability in real-time, and get answers from an AI persona specialized in Web3 tokenomics and the 0G ecosystem.

## Features

| | Feature | Description |
|---|---|---|
| 🤖 | AI Inference | Submit prompts via 0G Compute SDK or external LLM providers |

## Try It Instantly (No Setup Required)

Go to **[zero-cup.vercel.app](https://zero-cup.vercel.app)** and:

1. Click the **Settings** icon (top-right)
2. Paste a free [Groq](https://console.groq.com) API key in the format `groq=gsk_...`
3. Go to **Inference** and start asking tokenomics questions

To use **0G on-chain inference**, paste a wallet private key (with OG tokens on testnet) in Settings.

## Local Development

### Prerequisites

- Node.js 20+
- pnpm 10+
- PostgreSQL 15+ (or a free [Neon](https://neon.tech) cloud database)

### 1. Clone & install

```bash
git clone https://github.com/daikiijax/zero-cup.git
cd zero-cup
pnpm install
```

### 2. Configure environment

Create `.env` in the repo root:

```env
DATABASE_URL=postgresql://user:pass@host/db?sslmode=require
SESSION_SECRET=any-long-random-string

# Optional — 0G Network (defaults to testnet if omitted)
ZG_RPC_URL=https://evmrpc-testnet.0g.ai
ZG_SERVICE_URL=https://indexer-storage-testnet-standard.0g.ai
ZG_PRIVATE_KEY=0x...

# Optional — External LLM keys (format: provider=apikey)
LLM_API_KEY=groq=gsk_...
```

### 3. Push database schema

```bash
pnpm --filter @workspace/db run push
```

### 4. Start development servers (two terminals)

```bash
# Terminal 1 — API server on port 5000
PORT=5000 pnpm --filter @workspace/api-server run dev

# Terminal 2 — Frontend on port 5173
PORT=5173 BASE_PATH=/ pnpm --filter @workspace/compute-dashboard run dev
```

Open `http://localhost:5173` — frontend proxies API calls to `:5000` automatically.

## Deploy to Vercel

### 1. Create a Postgres database

[Neon](https://neon.tech) has a generous free tier. Copy the **pooler** connection string ending in `?sslmode=require`.

### 2. Push the DB schema

```bash
DATABASE_URL=your-connection-string pnpm --filter @workspace/db run push
```

### 3. Import the repo to Vercel

1. Go to [vercel.com/new](https://vercel.com/new) → **Import Git Repository** → select `zero-cup`
2. Set **Framework Preset** to *Other* — `vercel.json` handles routing
3. Add **Environment Variables**:

| Variable | Required | Value |
|---|---|---|
| `DATABASE_URL` | ✅ | Postgres connection string (`?sslmode=require`) |
| `SESSION_SECRET` | ✅ | Any long random string |
| `ZG_RPC_URL` | — | Defaults to testnet if omitted |
| `ZG_PRIVATE_KEY` | — | Wallet private key for on-chain inference |
| `LLM_API_KEY` | — | `provider=apikey` (e.g. `groq=gsk_...`) |

4. Click **Deploy**

Frontend and API serverless function deploy together under one `.vercel.app` domain.

### 4. Verify

```bash
curl https://your-app.vercel.app/api/healthz
# {"status":"ok"}
```

## External LLM Key Format

`LLM_API_KEY` accepts `provider=apikey`. Multiple providers can be comma-separated.

```
groq=gsk_...          # Groq (free tier available)
openai=sk-...         # OpenAI
openrouter=sk-or-...  # OpenRouter
anthropic=sk-ant-...  # Anthropic
deepseek=sk-...       # DeepSeek
mistral=...           # Mistral
xai=xai-...           # xAI / Grok
0g=your-key           # 0G Private Computer (router-api.0g.ai)
```

Users can also set keys directly in the browser via **Settings** — stored in `localStorage`, never sent to the server.
